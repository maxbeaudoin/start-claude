# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
bun run dev          # dev server on port 3000
bun run build        # production build → .output/
bun run start        # serve production build
bun run test         # run all tests; append a file path to run one
bun run typecheck    # TypeScript type checking
bun run check:fix    # Biome lint + format (auto-fix)
bun run test:e2e     # Playwright E2E tests (requires dev server or uses webServer config)
bunx shadcn add <component>
```

## Task Workflow

The primary entry point is `/ship`. The pipeline skills exist for re-running individual steps.

- **`/ship <Linear issue ID or description>`** — Full pipeline: fetches context, creates branch, runs the appropriate skills, pushes, opens PR, posts to Linear, waits for Copilot review. Skips spec/plan automatically for bug fixes, docs, and chores.
- **`/review <PR number>`** — Standalone: evaluates Copilot review comments on any PR, implements valuable ones, dismisses weak ones with rationale.

**Pipeline skills** (called by `/ship`, or individually to re-run a step):
- **`/spec`** — Writes or updates `specs/<feature>/spec.md`. Reads from `.claude/context.md` or `$ARGUMENTS`.
- **`/plan`** — Reads spec changes on the current branch (`git diff main -- specs/`), writes `changes/<slug>/plan.md`. Optional.
- **`/code`** — Implements new scenarios from the spec diff, writes tests, runs quality checks, commits.

**Artifacts**:
- Specs: `specs/<feature>/spec.md` — living capability docs, per-feature, never deleted
- Plans: `changes/<issue-id>-<slug>/plan.md` — versioned planning artifacts, committed
- ADRs: `docs/adr/NNNN-<slug>.md` (MADR v3 format) — architectural decisions, when warranted
- E2E tests: `e2e/<feature>.spec.ts` (Playwright) — written at `/code` time from spec scenarios
- Unit tests: `src/features/<feature>/<name>.test.ts` (Vitest) — co-located with feature code

## Git Workflow

Branches: `feat/`, `fix/`, `chore/`, `docs/` + short description.
Commits follow Conventional Commits — `feat:`, `fix:`, `chore:`, `docs:`, `refactor:`, `test:`.

## Principles

- **KISS** — prefer the simplest solution that works
- **YAGNI** — don't build for hypothetical future needs
- **POLA** — code should behave as a reader would expect, avoid surprises
- **DRY** — extract shared logic, but only once there are 3+ real uses
- **SOLID** — single responsibility, open/closed, Liskov substitution, interface segregation, dependency inversion

## Testing

**E2E (Playwright)** — spec scenarios involving UI rendering or user interaction. Config: `playwright.config.ts`. Run with `bun run test:e2e`. Lives in `e2e/<feature>.spec.ts`.

SSR hydration patterns to follow in every Playwright test:
- Call `await page.waitForLoadState("networkidle")` after every `page.goto()` — the SSR HTML arrives before React hydrates, so interactions fired immediately will miss event handlers
- Use `pressSequentially("text")` instead of `fill("text")` for React controlled inputs — `fill()` fires native DOM events that bypass React's synthetic event system

**Unit (Vitest)** — spec scenarios covering server functions, business logic, or data transforms. Co-located with feature code: `src/features/<feature>/<name>.test.ts`. Config: `vitest.config.ts` (excludes `e2e/`). Run with `bun run test`.

## Architecture

**TanStack Start** full-stack React app — file-based routing, SSR, server functions. Vite + Nitro build outputs to `.output/`.

**Feature folders** (`src/features/`): Business logic is organized by capability. Each feature owns its components, server functions, hooks, types, and unit tests. See `docs/adr/0001-feature-folders.md`.

**Routing** (`src/routes/`): Thin routing layer required by TanStack Start. Route files import from `src/features/` and contain no business logic. `__root.tsx` is the root layout (QueryClientProvider, Header, devtools). `api.*.ts` files define API handlers. `routeTree.gen.ts` is auto-generated by `@tanstack/router-plugin` — do not edit.

**Shared utilities** (`src/lib/`): Code with no single feature owner.

**Styling**: Tailwind CSS v4. Global CSS in `src/styles.css`. shadcn/ui uses "new-york" style, `neutral` base color, CSS variables, Lucide icons.

**Path alias**: `@/*` → `src/*`.

**Linting**: Biome (not ESLint/Prettier) — tabs, double quotes. `src/routeTree.gen.ts` excluded.
